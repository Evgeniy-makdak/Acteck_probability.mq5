# Acteck Probability MT5

Полуавтоматический советник для MetaTrader 5, который анализирует волатильность и вероятности коррекционного движения по алгоритму ACTECK.

Проект адаптирован под MT5 и оптимизирован по UI/производительности:
- нет тяжелой мультивалютной таблицы на весь экран;
- в каждой вкладке графика отображаются данные только для текущего символа;
- компактная панель с режимами `Trend / Duration / Speed`;
- автоскрытие панели для чистой визуализации паттернов.

---

## 1) Что находится в проекте

Структура:

- `Experts/Acteck_probability/Acteck_probability.mq5`
  - главный файл советника (точка входа);
  - UI-панель, обработка событий, таймер обновления;
  - переключение режимов;
  - экспорт CSV-отчетов.
- `Include/Acteck_probability/TrendEngine.mqh`
  - ядро алгоритма поиска трендов/экстремумов;
  - расчет исторических движений;
  - вероятности и производные метрики;
  - калибровка pip-size для 3/5 знаков и остальных инструментов.

## 2) Быстрая установка (для самых неопытных)

### Шаг 1. Откройте папку данных MT5
1. Запустите терминал **MetaTrader 5**.
2. В верхнем меню нажмите **File -> Open Data Folder**.

### Шаг 2. Скопируйте файлы проекта в MQL5
В открывшейся папке Data Folder зайдите в `MQL5` и скопируйте:

- файл `Acteck_probability.mq5` в папку:
  - `MQL5/Experts/Acteck_probability/`
- файл `TrendEngine.mqh` в папку:
  - `MQL5/Include/Acteck_probability/`

Если подпапок `Acteck_probability` нет, создайте их вручную.

### Шаг 3. Скомпилируйте советник
1. В MT5 откройте **MetaEditor** (клавиша `F4`).
2. В дереве файлов откройте:
   - `Experts/Acteck_probability/Acteck_probability.mq5`
3. Нажмите **Compile** (`F7`).
4. Убедитесь, что внизу нет ошибок компиляции.

### Шаг 4. Установите советник на график
1. Вернитесь в MT5.
2. Откройте нужный символ (например `EURUSD`) и таймфрейм.
3. Перетащите из Navigator советник `Acteck_probability` на график.
4. Разрешите запуск советника в настройках (галочки Algo Trading/Allow).

Готово: советник начнет рассчитывать значения и рисовать уровни.

---

## 3) Куда именно класть файлы (шпаргалка)

- `Acteck_probability.mq5` -> `.../MQL5/Experts/Acteck_probability/`
- `TrendEngine.mqh` -> `.../MQL5/Include/Acteck_probability/`

Нельзя класть `TrendEngine.mqh` в `Experts`, иначе не сработает include.

## 4) Мини-курс по использованию советника

### 4.1 Логика работы
- Советник берет исторические свечи с `InpStartDate` до текущего времени.
- Находит направленные движения, где откат меньше фильтра “поглощается”, а откат >= фильтра считается сменой направления.
- Для текущего незавершенного движения рассчитывает:
  - вероятность коррекции;
  - текущий ход в пипсах;
  - исторические контрольные значения.

### 4.2 Режимы панели
- `Trend`
  - классический вероятностный режим по величине движения в пипсах;
  - рисует сетку уровней вероятности (лаймовые горизонтали).
- `Duration`
  - вероятность по длительности движения во времени;
  - сетка вероятностей на графике не рисуется (чтобы не перегружать).
- `Speed`
  - динамика скорости движения (`pips/min`) относительно средней исторической скорости.

Переключение режимов: кнопка **Mode** на панели.

### 4.3 Работа с фильтрами (5 столбцов)
У каждого столбца есть:
- свой фильтр пипсов (`InpLevel1Pips ... InpLevel5Pips`);
- свой таймфрейм (`InpTF1 ... InpTF5`).

Нажмите на нужную ячейку — график переключится на таймфрейм этого столбца.

### 4.4 Цвета и сигналы
- Если вероятность >= `InpAlertLevel`, ячейка подсвечивается:
  - синим — при одном направлении;
  - красным — при противоположном.
- При первом достижении порога звучит `alert.wav`.

### 4.5 Auto-hide
- Панель автоматически скрывается через `InpAutoHideSeconds` секунд бездействия.
- Кнопка `Hide/Show` позволяет скрывать/показывать панель вручную.

## 5) Входные параметры (простым языком)

- `InpStartDate`  
  С какой даты брать историю для статистики.

- `InpUpdateSeconds`  
  Как часто обновлять расчеты и UI.

- `InpAlertLevel`  
  Порог вероятности для подсветки и звука.

- `InpATRPeriod`  
  Период ATR для информационного значения в панели.

- `InpMinProbability`  
  Минимальная вероятность для рисования уровней в режиме Trend.

- `InpAutoHideSeconds`  
  Через сколько секунд прятать панель.

- `InpExportCsvOnInit`  
  Делать экспорт CSV на старте.

- `InpLevel1Pips ... InpLevel5Pips`  
  Фильтры волатильности (5 столбцов).

- `InpTF1 ... InpTF5`  
  Таймфреймы для соответствующих столбцов.

## 6) CSV-отчеты: где лежат и что внутри

Файлы пишутся в:
- `MQL5/Files/Acteck_probability/`

Формат имен:
- `<SYMBOL>_<TF>_<FILTER>_table1.csv`
- `<SYMBOL>_<TF>_<FILTER>_table2.csv`

`table1` содержит:
- номер;
- buy/sell;
- цены начала/конца;
- время начала/конца;
- длительность в минутах;
- размер движения в пипсах;
- скорость (`pips/min`).

`table2` содержит:
- значения по порядку;
- значения по убыванию;
- рассчитанную вероятность.

---

## 7) Важный момент про пипсы/пункты (3/5 знаков и другие активы)

В проекте реализована корректная калибровка:
- для 5 и 3 знаков: `1 pip = 10 points`;
- для остальных случаев: `1 pip = 1 point`.

Это нужно, чтобы значения были сопоставимы между FX, metals и crypto.

## 8) Типовой сценарий работы

1. Откройте `EURUSD`, повесьте советник.
2. Поставьте `InpStartDate` под нужный горизонт истории (например, 2010+).
3. Оставьте 3-5 фильтров (например 500/1000/1500/2000/2500).
4. Работайте в `Trend` для вероятностей разворота по ходу цены.
5. Переключайтесь в `Duration`, если важно время фазы.
6. Используйте `Speed`, когда нужно понять ускорение/затухание движения.
7. По сигналам >= `InpAlertLevel` проверяйте подтверждение паттерном цены.

---

## 9) Если что-то не работает

- Не компилируется:
  - проверьте, что `TrendEngine.mqh` лежит именно в `Include/Acteck_probability/`.
- Нет данных:
  - проверьте загрузку истории по символу/таймфрейму.
- Нет кнопок/панели:
  - включите Algo Trading и убедитесь, что советник запущен.
- Нет CSV:
  - проверьте папку `MQL5/Files/Acteck_probability/` и права записи.

## 10) Дисклеймер

Советник предоставляет статистические вероятности и визуальные сигналы, но не гарантирует результат сделок. Используйте риск-менеджмент и тестируйте параметры на демо/истории перед реальной торговлей.

## 11) На каких таймфреймах лучше применять

Проведен отдельный прогон логики советника на истории по основным парам:
- `EURUSD`
- `GBPUSD`
- `USDCHF`
- `USDJPY`

Методика:
- исторические данные: последние 2 года;
- базовая серия: H1, далее ресемплинг в H4 и D1;
- фильтры: 50/100/150/200/250 пипсов;
- оценка: комбинированный score (достаточность выборки + контраст вероятностей + стабильность скорости движения).

Итог по каждой паре:
- `EURUSD`: D1 (лучше), затем H4, затем H1.
- `GBPUSD`: D1 (лучше), затем H4, затем H1.
- `USDCHF`: D1 (лучше), H4 и H1 почти равны.
- `USDJPY`: D1 (лучше), затем H4, затем H1.

Сводный рейтинг:
- `D1`: лучший средний score.
- `H4`: второй, хороший компромисс между частотой и качеством.
- `H1`: самый частый по сигналам, но более шумный.

Практическая рекомендация:
- основной рабочий ТФ: **D1**;
- для более частого мониторинга: **H4**;
- `H1` использовать как вспомогательный, только при дополнительной фильтрации (контекст старшего ТФ, новостной фильтр, контроль волатильности).

Файл с подробными числовыми результатами:
- `analysis_timeframes.md`
